# v0.7.1 Roadmap

## 版本定位

v0.7.1 是 v0.7.0 之后的小版本修正，聚焦一个问题：网络代理配置需要在应用内所有外部联网请求中保持一致。

v0.7.0 已经支持 GitHub API、精选 Skills、GitHub Contents 下载和 Git 更新走代理，但应用自更新、更新说明拉取和 Explore 在线搜索仍存在绕过应用代理配置的路径。v0.7.1 的目标是收敛这些路径，让用户开启网络代理后获得一致、可预期的联网行为。

## 版本目标

- 网络代理从“GitHub 相关代理”升级为“应用网络代理”。
- 应用自更新检查、下载和安装使用同一代理配置。
- 更新弹窗中的 release notes 拉取使用同一代理配置。
- Explore 在线搜索使用同一代理配置。
- 关闭代理后，所有外部联网请求回到默认网络行为。

## 模块总览

| 模块 | 状态 | 说明 | 文档 |
| --- | --- | --- | --- |
| 统一网络代理 | 已实现 | 开启网络代理后，应用内外部 HTTP、Git 和更新请求统一使用代理入口 | [unified-network-proxy.md](unified-network-proxy.md) |

## 验收标准

- 开启网络代理后，应用更新检查使用当前代理配置。
- 开启网络代理后，应用更新下载和安装使用当前代理配置。
- 开启网络代理后，更新弹窗中的 release notes 通过后端代理请求获取。
- 开启网络代理后，Explore 在线搜索通过当前代理请求 `skills.sh`。
- 已有 GitHub 搜索、精选 Skills、GitHub Contents 下载和 Git clone/fetch/update 代理能力不回退。
- 关闭网络代理后，上述请求不强制使用代理。
- 设置页文案不再暗示只影响 GitHub，而是表达为应用联网请求的统一代理入口。
