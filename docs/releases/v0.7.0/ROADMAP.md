# v0.7.0 Roadmap

## 版本定位

v0.7.0 的核心方向是完善 Skills Hub 的多工具同步管理体验。

本版本聚焦已经验证过的高价值能力：安装范围选择、批量管理、批量标签整理、Skill 启停、自动更新体验优化、工具管理、自定义工具目录和网络代理。版本目标是让用户安装、同步、更新和配置工具时更稳定、更可控，同时减少重复操作。

## 版本目标

- 安装 Skill 时即可选择全局或项目范围，减少安装后的重复调整。
- 在首页完成批量选择、批量标签整理、批量工具设置和批量删除，减少逐个操作。
- 支持单个和批量启用/停用 Skill，让用户可以临时暂停 Skill 生效而不删除配置。
- 自动更新从手动能力升级为可配置、可观察、可后台运行的更新机制。
- 工具列表从固定检测升级为可管理、可关闭、可扩展的工具配置。
- GitHub 访问能力适配更复杂的网络环境，降低搜索、安装和更新失败率。

## 验收标准

- 用户可以在安装流程中选择全局或项目同步范围。
- 用户可以进入批量管理模式，选择当前结果或单独选择多个 Skills。
- 用户可以对所选 Skills 批量添加或移除标签，并可在操作中创建新标签。
- 用户可以对所选 Skills 统一设置目标工具；工具设置以当前勾选结果为准。
- 用户可以批量删除所选 Skills，且不会影响未选中的 Skills。
- 用户可以单个或批量停用 Skills；停用后目标工具不再读取这些 Skills。
- 用户可以重新启用已停用 Skills，并按停用前的工具设置恢复同步。
- 用户可以配置自动更新，并查看后台更新的进度和失败原因。
- 用户可以管理内置工具启用状态，并添加自定义工具目录。
- GitHub 搜索、安装、精选 Skills 和更新流程可以使用统一代理配置。
- 所有已完成模块的中英文文案完整，前后端检查通过。
