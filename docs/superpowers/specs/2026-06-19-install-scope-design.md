# 安装时选择全局或项目范围

## 背景

Skills Hub 已支持在 Skill 安装完成后，通过“同步范围”弹窗在全局和项目范围之间切换。但手动安装或从仓库批量安装多个 Skill 时，当前流程只允许选择目标工具，所有 Skill 会先按全局范围同步。用户若希望项目级安装，只能在安装后逐个调整，批量操作成本较高。

## 目标

- 在安装前一次选择本批 Skill 的同步范围。
- 默认保持全局安装，避免改变现有用户习惯。
- 项目范围支持一次选择多个项目目录。
- 批量发现多个 Skill 时，同一范围、项目目录和工具配置应用于本批所有 Skill。
- 复用现有项目范围能力，不引入第二套范围模型或数据库结构。

## 非目标

- 不改变安装完成后的单个 Skill 范围调整能力。
- 不允许同一批 Skill 使用不同的范围配置。
- 不为不支持项目范围的工具增加新的后端适配能力。
- 不重构与本功能无关的安装流程。

## 页面与交互

在新增 Skill 弹窗的“安装到工具”区域上方增加“安装范围”区域，信息顺序为：

1. Skill 来源、名称和标签。
2. 安装范围。
3. 目标工具。

安装范围使用现有“同步范围”弹窗中的全局/项目选择样式：

- **全局**：默认选中，说明为“在所有项目中可用”。
- **项目**：说明为“仅在选择的项目中可用”。

切换到项目范围后，展开项目目录选择区域：

- 展示已选择的项目目录，并允许逐项移除。
- 提供“选择项目目录”按钮。
- 展示未被选中的最近项目，支持快捷添加。
- 对项目路径去重。
- 未选择任何项目目录时显示现有必填提示，并禁用创建或安装按钮。

项目范围下，不支持项目范围的工具自动取消选中并置灰，同时显示“不支持项目安装”的原因。切回全局范围时不自动恢复这些工具，避免隐式改变用户选择。

安装流程识别出多个 Skill 并进入批量选择界面后，继续使用首次提交时的安装范围、项目目录和目标工具配置，不要求用户逐个配置。

安装完成后，用户仍可通过 Skill 卡片上的现有“同步范围”入口单独调整。

## 组件设计

从现有 `ScopeSyncModal` 中抽取共享的范围选择展示组件。该组件只负责：

- 展示和切换 `global` / `project`。
- 展示项目目录列表。
- 添加、移除项目目录。
- 展示最近项目。
- 展示项目目录必填状态。

`ScopeSyncModal` 继续负责单个已安装 Skill 的应用按钮和范围变更流程；`AddSkillModal` 负责安装表单提交条件、工具可用状态和批量安装配置。共享组件不直接调用 Tauri 命令，所有状态和事件通过 props 传入。

## 状态与数据流

`App.tsx` 增加安装流程状态：

```ts
installScope: 'global' | 'project'
installProjects: string[]
```

行为规则：

- 打开新增 Skill 弹窗时，`installScope` 初始化为 `global`，`installProjects` 初始化为空数组。
- 关闭或成功完成安装后重置这两个状态。
- 选择项目目录后调用现有 `save_recent_project`，并刷新最近项目列表。
- 项目目录选择复用现有目录选择能力。
- 项目模式下的可选工具根据 `ToolOption.supports_project_scope` 计算。

安装 Skill 到中央仓库的现有命令保持不变。Skill 创建成功后继续调用现有 `sync_skill_to_tool`：

- 全局范围：每个 `Skill × 工具` 调用一次，传入 `scope: 'global'`。
- 项目范围：每个 `Skill × 工具 × 项目目录` 调用一次，传入 `scope: 'project'` 和对应的 `projectPath`。

同一物理 skills 目录对应多个工具时，继续使用现有共享目录去重和后端记录逻辑。

## 批量安装

本设计覆盖本地目录和 Git 仓库两类批量发现流程：

- 用户在新增 Skill 弹窗中选择一次范围、项目和工具。
- 若发现多个 Skill，候选选择弹窗只负责选择要安装的 Skill。
- 确认候选后，所有选中的 Skill 使用此前保存的安装配置。
- 安装过程中某个同步任务失败，不中断其余 Skill、工具或项目的处理。
- 完成后沿用现有错误汇总方式展示失败项。

## 校验与错误处理

- 项目范围且项目目录为空：阻止提交并显示现有项目必填文案。
- 项目范围且所有工具均不可用或未选择：沿用“未选择同步目标”错误。
- 项目路径不存在：沿用后端 `projectPath must be an existing directory` 错误。
- 工具不支持项目范围：前端阻止选择，后端 `PROJECT_SCOPE_UNSUPPORTED` 继续作为保护。
- 目录无写入权限、目标已存在和其他同步错误：沿用现有错误格式及汇总展示。
- 部分任务失败：继续执行剩余任务，安装到中央仓库成功的 Skill 保持已安装状态。

## 国际化

所有新增可见文本加入 `src/i18n/resources.ts`，同时提供英文和中文。优先复用现有 `scope.*` 和 `projectSync.*` 文案，只为安装场景中缺失的标题、说明和工具禁用原因新增键。

## 测试与验证

前端测试或可测试逻辑应覆盖：

- 新增 Skill 弹窗默认选择全局范围。
- 全局和项目范围切换。
- 项目范围未选择目录时禁止提交。
- 多个项目目录的添加、去重和移除。
- 切换项目范围后，不支持项目范围的工具被取消并禁用。
- 切回全局后不自动恢复被取消的工具。
- 单个 Skill 的全局同步参数。
- 单个和批量 Skill 的多项目同步调用组合。
- 某个同步调用失败后继续处理剩余任务并汇总错误。

实现完成后必须运行：

```bash
npm run check
```
